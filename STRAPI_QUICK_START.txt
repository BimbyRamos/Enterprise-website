If you get an error when clicking "Save" after creating fields:

COMMON CAUSES:
1. Enumeration values with spaces or special characters
2. Field names using reserved words (id, createdAt, etc.)
3. Too many fields added at once
4. Database connection issues

QUICK FIXES:

âœ“ For Enumeration Fields (Status, Category):
  - Use single words: "Completed", "InProgress", "Planned"
  - NO spaces: âŒ "In Progress" â†’ âœ… "InProgress"
  - NO special chars: âŒ "Web Dev 2.0" â†’ âœ… "WebDevelopment"
  - Alternative: Use Text field instead of Enumeration

âœ“ For Field Names:
  - Use camelCase: featuredImage, publishedAt
  - Avoid: id, createdAt, updatedAt (reserved)
  - No special characters or spaces

âœ“ If Still Failing:
  1. Check terminal for error messages
  2. Add fields ONE AT A TIME
  3. Click "Save" after each field
  4. Restart Strapi: Ctrl+C, then npm run develop
  5. Clear browser cache (Ctrl+Shift+Delete)
  6. Try incognito mode

âœ“ For JSON Fields:
  1. Click "Add another field"
  2. Select "JSON" (not Text)
  3. Name it: technologies, metrics, features, etc.
  4. Click "Finish"
  5. Click "Save"
  
  When adding content, use proper JSON format:
  - Array: ["item1", "item2", "item3"]
  - Objects: [{"label": "Name", "value": "99%"}]
  - Use double quotes, not single quotes
  - No trailing commas
